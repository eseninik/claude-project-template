# Update: Smart Decision Making for Plan Execution

## Context

В template проекте добавлена умная логика принятия решений для выбора между параллельным и последовательным выполнением планов.

## Что изменилось

### 1. Логика принятия решений (CLAUDE.md)

**Раньше:**
- План с зависимостями → сразу executing-plans (последовательно)
- Агент спрашивал пользователя что выбрать

**Теперь:**
- Агент **сам анализирует** зависимости и волновую структуру
- Различает **волновую параллелизацию** vs **строгую последовательность**
- **Волновая структура** (Wave 1: [1,2] параллельно, Wave 2: [3]) → tasks/*.md
- **Строгая последовательность** (все waves = 1 task) → executing-plans
- Спрашивает пользователя только при низкой уверенности (<70%)

### 2. User preference (CLAUDE.md)

Добавлен приоритет пользователя:
```
User preference: Приоритет на скорость и параллелизм
```

При выборе подхода агент ВСЕГДА предпочитает параллельное выполнение, если задачи независимы или можно распараллелить по волнам.

### 3. Новый guide (dependency-analysis.md)

Создан алгоритм анализа зависимостей:
- Определение независимости между задачами
- Построение волновой структуры (`build_waves()`)
- Выбор стратегии (`choose_strategy()`)
- Примеры с реальными кейсами


   ```markdown

   ```

3. **Обнови AUTO-CHECK вариант B в CLAUDE.md:**
