### Key learnings

- **Watchdog должен иметь state между вызовами.** Без этого любой triggering ответ входит в infinite loop.
- **Severity должна быть graded.** Один канал (exit 2) превращает советы в приказы.
- **Task-class matters.** Один и тот же хук должен работать по-разному для chat/bugfix/deploy.
- **Narrow keywords.** Слова "bug/error/ошибк" слишком частотные — они не отличают нормальную дискуссию от реальной аномалии.
- **Codex иногда сам себя ловит.** latest.json содержал его же review собственной over-blocking реализации. Нашёл это случайно — полезный сигнал.
