# GOAL-BACKWARD VERIFICATION

**Проверяй от цели, не от задач.**

## Метод

```
1. Что должно быть ИСТИНОЙ? → Наблюдаемые поведения
2. Что должно СУЩЕСТВОВАТЬ? → Файлы, функции
3. Что должно быть СВЯЗАНО? → Импорты, вызовы
4. Проверь КАЖДЫЙ уровень
```

**Артефакт существует но не подключён = цель НЕ достигнута.**

---

## Уровни верификации

### Уровень 1: Существование
- Файл создан?
- Функция определена?
- Тест написан?

### Уровень 2: Связность
- Импорты добавлены?
- Функция вызывается?
- Тест запускается?

### Уровень 3: Поведение
- Функция работает корректно?
- Тест проходит?
- Требования выполнены?

### Уровень 4: Интеграция
- Компонент интегрирован в систему?
- Другие части системы используют его?
- End-to-end сценарий работает?

---

## Когда использовать

**Загрузи этот файл при:**
- Сложной верификации с множеством зависимостей
- Подозрении что "работает локально, но не интегрировано"
- Необходимости проверить цепочку зависимостей

**Основная верификация:** Используй `verification-before-completion/SKILL.md`
**Сложные случаи:** Этот протокол
