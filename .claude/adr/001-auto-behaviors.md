# ADR-001: AUTO-BEHAVIORS вместо команд

**Date:** 2026-01-29
**Status:** Accepted

---

## Context

Пользователь хотел систему, которая работает автоматически без ручных команд:
- Не хочет вызывать `/resume`, `/project-context` и т.д.
- Хочет чтобы агент сам понимал когда загружать контекст
- Хочет чтобы память обновлялась автоматически

Deep research показал, что "project amnesia" — главная проблема AI-агентов.

## Decision

Заменить систему команд на AUTO-BEHAVIORS — триггерные правила в CLAUDE.md, которые агент выполняет автоматически.

## Rationale

- Команды требуют ручного вызова — пользователи забывают
- Триггеры в CLAUDE.md читаются автоматически при старте сессии
- Агент сам решает когда применять правило по триггеру
- Zero-friction workflow — пользователь только даёт задачи

## Alternatives Considered

| Option | Pros | Cons | Why Rejected |
|--------|------|------|--------------|
| Slash-команды | Явный контроль | Требуют ручного вызова | Friction, забывают |
| Background skills | Работают автоматически | Сложнее поддерживать | Оверинжиниринг |
| MCP сервер | Полная автоматизация | Требует инфраструктуры | Слишком сложно |

## Consequences

### Positive
- Нулевое участие пользователя в рутине
- Контекст всегда актуален
- Память обновляется автоматически

### Negative
- Больше токенов на чтение файлов при старте
- Агент может читать ненужные файлы

### Trade-offs
- Больше автоматизации = меньше контроля пользователя
- Приемлемо для данного use case

## Implementation Notes

- AUTO-BEHAVIORS секция в CLAUDE.md
- Триггеры: SESSION START, BEFORE CODE CHANGES, AFTER TASK COMPLETION, и т.д.
- Каждый триггер имеет чёткие условия и действия

## For AI Agents

- Before changing: Проверь что новый подход сохраняет автоматизацию
- If proposing alternative: Объясни как это не увеличит friction для пользователя
