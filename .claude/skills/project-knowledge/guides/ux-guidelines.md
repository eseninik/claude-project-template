# UX Guidelines

## Назначение
Правила для пользовательского интерфейса, текстов и взаимодействия.

---

## Язык интерфейса

**Основной язык:** Русский

**Исключения:**
- Технические термины без перевода (API, ID, etc.)
- Английские названия продуктов/сервисов

---

## Tone of Voice

**Как мы общаемся:**
- Дружелюбно, но профессионально
- Без канцелярита и сложных конструкций
- Кратко и по делу
- Уважительно (на "вы" или на "ты" - выбрать)

**Примеры:**
```
Хорошо: "Готово! Ваш заказ оформлен."
Плохо: "Операция по оформлению заказа успешно завершена."

Хорошо: "Что-то пошло не так. Попробуйте позже."
Плохо: "Произошла непредвиденная ошибка системы."
```

## Текстовые паттерны

### Кнопки
```
Действие: "Оформить заказ", "Отменить", "Назад"
Подтверждение: "Да, удалить", "Нет, оставить"
```

### Сообщения об ошибках
```
Формат: [Что случилось]. [Что делать].
Пример: "Не удалось загрузить данные. Попробуйте обновить страницу."
```

### Успешные действия
```
Формат: [Что произошло] + эмодзи (опционально)
Пример: "Настройки сохранены"
```

### Пустые состояния
```
Формат: [Что пусто]. [Как заполнить].
Пример: "У вас пока нет заказов. Выберите товар в каталоге."
```

## Design System (если есть UI)

### Цвета
- Primary: `#[hex]`
- Secondary: `#[hex]`
- Error: `#[hex]`
- Success: `#[hex]`

### Типографика
- Заголовки: [шрифт, размер]
- Текст: [шрифт, размер]

---

## Emoji Guidelines

**Когда использовать:**
- Успешные действия: ✅
- Ошибки: ❌
- Предупреждения: ⚠️
- Информация: ℹ️

**Когда НЕ использовать:**
- В формальных сообщениях
- В сообщениях об ошибках (кроме ❌)
- Подряд несколько emoji

---

## Accessibility

**Базовые правила:**
- Достаточный контраст текста
- Понятные подписи к кнопкам
- Альтернативный текст для изображений
- Не полагаться только на цвет для передачи информации
