# История изменений Roadmap

> Лог всех значительных изменений в планировании проекта.
> Помогает понять почему текущий план такой какой есть.

---

## Формат записи

```markdown
## [Дата] - [Тип изменения]

**Что изменилось:** [Описание]
**Почему:** [Причина/контекст]
**Влияние:** [Как это влияет на план]
**Решение принял:** [Кто]
```

---

## Типы изменений

| Тип | Описание | Пример |
|-----|----------|--------|
| `scope-change` | Добавлена/убрана фича | Добавили OAuth |
| `priority-change` | Изменён приоритет | P2 → P0 |
| `milestone-complete` | Milestone завершён | v1.0 released |
| `milestone-added` | Новый milestone | Добавили v1.1 |
| `timeline-change` | Изменились сроки | Сдвиг на 2 недели |
| `pivot` | Крупное изменение направления | Смена целевой аудитории |
| `blocker-resolved` | Блокер устранён | API доступен |
| `dependency-change` | Изменились зависимости | Новая библиотека |

## Как использовать этот файл

### При планировании
- Посмотреть историю чтобы понять контекст решений
- Не повторять ошибки прошлого
- Понять почему что-то "не в скоупе"

### При онбординге
- Быстро понять эволюцию проекта
- Понять текущие приоритеты и их причины

### При ретроспективе
- Оценить accuracy планирования
- Выявить паттерны (что часто меняется)
