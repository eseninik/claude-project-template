## Правила работы с milestones

### Когда создавать новый milestone
- В начале проекта (первый milestone)
- После завершения текущего
- При крупном pivot (переопределение скоупа)

### Когда обновлять этот документ
- При изменении статуса фичи
- При выполнении критерия выхода
- При обнаружении блокера
- При завершении milestone

### Хороший критерий выхода
✅ "Пользователь может зарегистрироваться через email"
✅ "API отвечает за < 200ms на 95 percentile"
❌ "Регистрация работает" (неконкретно)
❌ "Код качественный" (субъективно)

### Связь с roadmap.md
- `roadmap.md` - high-level план, редко меняется
- `milestones.md` - текущий milestone, обновляется часто
- `roadmap-history.md` - лог изменений
